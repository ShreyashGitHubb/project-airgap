Product Requirements Document: Antigravity — Zero-Trust Personal Medical and Diagnostic Analysis app built on google-ai-edge/gallery

PRD · v1.0 · Confidential

Antigravity
Zero-Trust Health Agent

A fully offline, on-device personal medical companion for symptom analysis, document translation, and diagnostic support — built on the google-ai-edge/gallery foundation.

Platform: Android (API 26+) Base: google-ai-edge/gallery Model: Gemma 3 / PaliGemma 2 Horizon: 12 months ✦ Zero cloud dependency
01 Problem statement

Trust is the missing feature

People desperately want AI to help them understand their symptoms and decipher opaque medical paperwork — but they will not upload photos of their skin, their eyes, or their private lab reports to a startup's server. The fear is rational and the bottleneck is real. Existing health apps either demand cloud connectivity (destroying trust) or offer only superficial chatbot experiences (destroying utility). Antigravity eliminates the dichotomy entirely: every byte of data is processed locally, on hardware the user already owns and controls.

02 Strategic pillars

Four promises, kept on-device

🔒
Radical privacy

No server. No account. No telemetry. The model, the data, and the inference all live on the user's phone.

👁
Multimodal perception

On-device vision understands skin photos, eye images, handwritten prescriptions, and multi-page lab reports — natively.

🧠
Explainable reasoning

Thinking Mode surfaces the model's step-by-step logic. Users see how conclusions were reached — not just what they are.

⚠️
Responsible guardrails

Structured outputs always end with a "see a professional" footer. The app informs; it never prescribes.

03 Repository leverage map

How gallery becomes Antigravity

Every feature is architected as an extension of an existing gallery capability — no forks, no rewrites.

google-ai-edge/gallery component → Antigravity usage
AskImage pipeline Multimodal triage — user photographs a skin lesion, eye, or physical document; gallery's vision pipeline parses it entirely on-device.
Agent Skills framework Specialized local skill modules: Skin Triage Skill, Eye Screen Skill, Prescription Translator Skill, Lab Report Decoder Skill.
Thinking Mode Exposes chain-of-thought reasoning on every medical analysis — users can audit the model's logic before acting on output.
Gemma 3n model loader Primary text + reasoning backbone. PaliGemma 2 loaded as second model slot for specialist vision tasks.
Offline vector store Bundled offline medical reference database (ICD-11 subset, drug interaction index) stored as on-device embeddings.
Summarize pipeline Powers the "Explain my report" workflow: user drops in a PDF or photo of a lab result and receives a plain-language summary.

06 Delivery roadmap

12-month phased plan

Phase 1 — Foundation Months 1–3
buildFork gallery, integrate Gemma 3n + PaliGemma 2 model loader. Implement zero-network policy (no internet permission). Build Prescription Translator and Lab Report Decoder skills. Basic Thinking Mode toggle.
Phase 2 — Vision Skills Months 4–6
buildSkin Triage Skill with structured ABCDE output. Eye Screen Skill (redness, discharge, structural anomalies). Image capture UX with guidance overlays. Bundled offline ICD-11 reference store.
testMedical accuracy review with licensed clinicians. Red-team adversarial image inputs.
Phase 3 — Agent & UX Polish Months 7–9
buildMulti-step agentic workflow: image → cross-reference offline DB → structured summary. Symptom history timeline (local encrypted storage). Emergency escalation screen (calls 112/999 — only outbound action permitted). testUsability study with 50 participants across literacy levels.
Phase 4 — Launch & Compliance Months 10–12
shipRegulatory classification review (CE Class I / FDA Software as Medical Device wellness tier). Privacy audit. Play Store listing emphasising zero-data architecture. Press & community launch.
07 Risks & mitigations

What could go wrong

Risk Level Mitigation
Model hallucinates a dangerous medical conclusion High Structured output schema enforces uncertainty ranges. All outputs append mandatory "consult a professional" notice. Clinical red-team testing in Phase 2.
On-device model too slow on low-end hardware Medium Ship a "lite" model profile (Gemma 2B) for devices with <6GB RAM. Stream progressive tokens to avoid perceived latency.
Regulatory misclassification as a medical device Medium Scope all claims to "educational awareness" not "diagnosis". Legal review before Phase 4 launch.
User acts on output instead of seeking care High Mandatory friction screen before analysis: "I understand this is not a medical diagnosis." Emergency escalation button always visible.
Storage footprint exceeds user tolerance Low Modular skill downloads — users install only the skills they need. Core app + one skill targets <1.5 GB.
This PRD is a product planning document, not a medical opinion. Antigravity does not diagnose disease. All feature descriptions involving medical inference require clinical review and appropriate regulatory classification before shipping.
